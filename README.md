# the-systematic-schemer



## The Philosophy



This repository represents a deliberate approach to learning functional programming. I am working through the book "The Little Schemer" by Daniel P. Friedman and Matthias Felleisen. I have enjoyed working through the book with pen and paper, but this time, rather than relying on intuition alone, I am applying the How to Design Programs (HtDP) methodology developed by Felleisen, Gregor Kiczales, and others.



## Why this approach?



* The Little Schemer is well-known for its Socratic method and building intuition about recursion and pattern matching.



* The HtDP Pedagogy (and Gregor Kiczales' Systematic Program Design) provides a rigorous design recipe. It uses a repeatable process: defining data, drafting templates based on that data, and using tests to validate correctness.



I believe that applying HtDP to the "Schemer" exercises is valuable because it turns abstract recursion into a concrete engineering discipline. It proves that even "mind-bending" problems become manageable when you follow a systematic recipe.



## How to Run



I am using Racket for this project. 



* Dr Racket: Open any file and click **Run**. I use module+ test with a text-based runner to provide feedback in the interactions window.
* Command Line: Run `raco test .`  from the root directory to execute the entire test suite.
